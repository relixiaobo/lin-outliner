# Tenon Design System Decision Audit

This audit proves whether Tenon's UI is generated by a small rule set instead of
page-local special cases. Each sampled decision must point to the smallest owning
rule: foundation token, state model, component contract, surface model, pattern,
or named exception.

Target: at least 80% of sampled UI decisions are `Derived`. `Exception` rows must
be named in the kernel Exception Registry.
